# 2026-06-15 관리자 페이지 신설 디자인 스펙

## 1. 개요
성만 FC 사이트의 뉴스, 선수단, 경기 일정 정보를 브라우저 환경에서 동적으로 관리할 수 있는 **클라이언트 사이드 관리자 페이지**를 신설합니다. 백엔드 데이터베이스 없이 **로컬 스토리지(localStorage)**를 활용하여 CRUD 기능을 지원하며, 푸터에 숨겨진 진입 링크를 제공합니다.

---

## 2. 보안 및 인증 흐름
* **임시 관리자 인증 정보**:
  * 아이디: `admin`
  * 비밀번호: `admin1234`
* **인증 상태 관리**:
  * `sessionStorage` 또는 `js/app.js` 내의 상태 변수 `isAdminLoggedIn`을 활용해 관리자 로그인 상태를 유지합니다.
* **라우팅 보안**:
  * 로그인하지 않은 사용자가 `#admin-dashboard`에 접근하면 자동으로 `#admin-login`으로 리다이렉트합니다.
  * 로그인한 관리자가 `#admin-login`에 접근하면 자동으로 `#admin-dashboard`로 리다이렉트합니다.

## 3. 네비게이션 및 진입 경로
* **푸터(Footer) 영역 진입 링크**:
  * [index.html](file:///J:/project/fcclub/index.html)의 `<footer>` 영역 내 저작권 표시 옆에 작은 회색 텍스트로 `| <a href="#admin-login" class="admin-link">Admin</a>` 링크를 신설합니다.
* **로그아웃 처리**:
  * 대시보드 내 로그아웃 버튼을 누르면 `isAdminLoggedIn`을 `false`로 설정하고 홈 화면(`#home`)으로 리다이렉트합니다.

---

## 4. 데이터 로컬 스토리지(localStorage) 마이그레이션
기존 `js/data.js`에 정의된 정적 데이터(`squadData`, `matchData`, `newsData`)를 로컬 스토리지에 동적 동기화할 수 있도록 통합 라이프사이클을 수정합니다:
* **초기화 시점 (`DOMContentLoaded`)**:
  * 로컬 스토리지에 `sungman_news`, `sungman_squad`, `sungman_matches`가 없으면 초기 mock 데이터를 직렬화(`JSON.stringify`)하여 로컬 스토리지에 탑재합니다.
  * 이후 모든 렌더링 로직(`bindNewsWidget`, `renderNewsPage`, `renderSquad`, `bindMatchCenter`)은 로컬 스토리지에서 데이터를 읽어와 파싱한 전역 상태 배열(`newsList`, `squadList`, `matchList`)을 사용해 화면을 그립니다.

## 5. 관리자 페이지 레이아웃
### A. 로그인 화면 (`#admin-login`)
* 깔끔한 글래스모피즘 스타일의 카드 레이아웃 양식.
* 아이디 및 비밀번호 입력 폼, 로그인 버튼.
* 로그인 실패 시 경고 메시지 텍스트 동적 표시.

### B. 관리자 대시보드 (`#admin-dashboard`)
* 팬존이나 뉴스 탭과 유사한 2단 분할 레이아웃 구성:
  * **좌측 메뉴 칼럼**:
    * 뉴스 관리 (News)
    * 선수단 관리 (Squad)
    * 경기 관리 (Matches)
    * 로그아웃 (Logout)
  * **우측 작업 칼럼**:
    * 선택한 관리 메뉴에 따라 동적으로 양식(목록 및 추가/수정 폼)이 렌더링됩니다.
    * **뉴스 관리**: 뉴스 목록 테이블/리스트, 기사 추가 버튼, 클릭 시 제목/날짜/본문을 입력하는 작성 및 수정 폼, 삭제 버튼.
    * **선수단 관리**: 선수 목록 테이블/리스트, 선수 추가 버튼, 클릭 시 이름/등번호/포지션/세부 스탯을 수정하는 폼, 삭제 버튼.
    * **경기 관리**: 경기 일정 목록, 경기 추가 버튼, 클릭 시 상대 팀/날짜/시간/장소/스코어 입력 폼, 삭제 버튼.

## 6. 접근성(a11y) 및 SEO 고려 사항
* 각 관리자 입력 필드(인풋, 텍스트에어리어)에 적절한 `<label>` 바인딩.
* 탭 간 이동 시 키보드 포커싱 제어 및 스크린 리더용 `aria-*` 속성 삽입.
* 데이터 삽입/수정 시 스크립트 인젝션 차단을 위한 완벽한 HTML 이스케이프(`escapeHTML`) 처리.
