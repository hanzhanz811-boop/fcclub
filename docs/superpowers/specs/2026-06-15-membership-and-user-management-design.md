# 2026-06-15 성만 FC 회원 가입 및 관리자 회원 관리 기능 연동 스펙

## 1. 개요
성만 FC 공식 홈페이지에 회원 가입 및 로그인 시스템을 도입하고, 관리자 페이지 내에 전체 회원 목록을 제어할 수 있는 회원 관리 기능을 신설합니다. 서버 백엔드 데이터베이스 없이 **로컬 스토리지(localStorage)**와 **세션 스토리지(sessionStorage)**를 활용하여 완벽한 회원제 클라이언트 사이드 프로토타입을 구축합니다.

---

## 2. 데이터 구조 및 스토리지 설계

### A. 회원 데이터베이스 (`localStorage.getItem('userData')`)
* **키:** `userData`
* **구조:** 회원 정보 객체들의 직렬화된 배열
* **초기 Mock 데이터 (최초 실행 시 자동 가입):**
  1. **관리자 계정:**
     - 이메일: `admin@sungmanfc.com`
     - 비밀번호: `admin1234`
     - 닉네임: `관리자`
     - 역할: `admin`
  2. **일반 회원 계정:**
     - 이메일: `user@sungmanfc.com`
     - 비밀번호: `user1234`
     - 닉네임: `성만팬`
     - 역할: `user`
* **회원 스키마:**
  ```json
  {
    "id": 1,
    "email": "user@sungmanfc.com",
    "password": "user1234",
    "nickname": "성만팬",
    "role": "user",
    "createdAt": "2026-06-15"
  }
  ```

### B. 로그인 세션 관리 (`sessionStorage.getItem('currentUser')`)
* **키:** `currentUser`
* **구조:** 로그인 성공 시, 비밀번호를 제외한 유저 정보 객체를 직렬화하여 세션 스토리지에 저장.
* **데이터 예시:**
  ```json
  {
    "id": 1,
    "email": "user@sungmanfc.com",
    "nickname": "성만팬",
    "role": "user",
    "createdAt": "2026-06-15"
  }
  ```

## 3. UI 및 라우팅 설계

### A. 로그인 화면 (`#login`)
* 독립된 해시 라우팅 페이지로 신설.
* 글래스모피즘 기반 로그인 카드 레이아웃.
* 이메일, 비밀번호 입력 필드 제공.
* 로그인 실패 시 에러 메시지 노출.
* 하단에 회원가입 페이지(`#signup`) 이동 링크 제공.

### B. 회원가입 화면 (`#signup`)
* 독립된 해시 라우팅 페이지로 신설.
* 이메일, 비밀번호, 비밀번호 확인, 닉네임 입력 필드 제공.
* 클라이언트 사이드 유효성 검증:
  * 이메일 형식 확인 (정규식 검증)
  * 비밀번호 및 비밀번호 확인 일치 체크
  * 이메일 및 닉네임 중복 여부 체크
* 가입 성공 시 로그인 페이지(`#login`)로 리다이렉트 및 안내 모달/메시지 노출.

### C. 동적 네비게이션 헤더 변경
현재 세션의 `currentUser` 여부에 따라 네비게이션 헤더의 메뉴를 동적으로 수정합니다.
1. **비회원 (로그아웃 상태):**
   - 메뉴 구성: Home, Club, Squad, Matches, News, Fan Zone, **로그인** (`#login`)
2. **일반 회원 로그인 상태:**
   - 메뉴 구성: Home, Club, Squad, Matches, News, Fan Zone, **로그아웃 (닉네임님)**
3. **관리자 로그인 상태:**
   - 메뉴 구성: Home, Club, Squad, Matches, News, Fan Zone, **관리자 대시보드** (`#admin-dashboard`), **로그아웃 (관리자님)**

## 4. 접근 제어 및 라우팅 가드 (Auth Guard)

* **비로그인 및 일반 회원의 관리자 화면 접근 통제:**
  * `#admin-dashboard` 접근 시 `currentUser`가 없거나 `currentUser.role !== 'admin'` 인 경우, 홈 화면(`#home`)으로 리다이렉트 후 경고 안내창("관리자 권한이 없습니다.") 노출.
* **로그인 회원의 인증 화면 접근 통제:**
  * 이미 로그인된 유저가 `#login` 또는 `#signup` 접근 시 홈 화면(`#home`)으로 즉시 리다이렉트.

---

## 5. 팬존 (커뮤니티) 연동 정책

* **작성 제한:**
  * 로그인하지 않은 상태에서 "글쓰기" 또는 "댓글 등록" 버튼 클릭 시, 로그인 화면(`#login`)으로 강제 리다이렉트하고 "로그인이 필요한 서비스입니다." 알림 노출.
* **작성자 매핑:**
  * 글 및 댓글 등록 시 수동 작성자 및 비밀번호 입력을 받지 않으며, `currentUser`의 정보가 작성자로 자동 귀속됩니다.
  * 게시글 저장 데이터에 `authorEmail` 또는 `userId` 필드를 연동하여 실제 회원이 쓴 글임을 입증합니다.
* **삭제 권한:**
  * 본인이 작성한 글/댓글인 경우에만 삭제 버튼 노출 및 즉시 삭제(비밀번호 검증 불필요).
  * 다른 사람의 글/댓글은 삭제 권한 제한 (삭제 버튼 숨김).
  * **관리자 예외 권한:** 관리자(Admin)는 전체 게시글과 댓글을 강제로 삭제할 수 있는 만능 권한(삭제 버튼 상시 활성화)을 부여받습니다.

## 6. 관리자 회원 관리 기능 (`#admin-dashboard` - Members Tab)

* **회원 관리 탭 추가:**
  * 관리자 대시보드 서브 메뉴에 `회원 관리` (`members`) 탭 추가.
* **회원 목록 테이블:**
  * 가입 회원 목록(가입일, 이메일, 닉네임, 권한) 렌더링.
* **관리 기능 구현:**
  * **권한 변경:** 특정 유저의 등급을 `user` ↔ `admin`으로 상호 변경 기능 제공.
  * **회원 탈퇴(삭제):** 특정 회원을 강퇴 처리.
  * **안전 장치:** 본인 계정은 강등 또는 탈퇴 처리가 불가능하도록 조작 방어 예외 처리.

## 7. 보안 및 접근성 (a11y & Security)
* 이메일, 닉네임 등 사용자 입력 데이터 렌더링 시 완벽한 스크립트 인젝션 방지를 위한 `escapeHTML` 처리 적용.
* 회원가입 및 로그인 폼 각 필드에 웹 접근성을 만족하는 `<label>` 명시 및 ARIA 오류 상태 추적 제공.
* 탭 전환 및 포커스 관리를 통해 키보드 접근성 유지.
