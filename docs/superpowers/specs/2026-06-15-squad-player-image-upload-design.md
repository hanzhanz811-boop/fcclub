# 2026-06-15 선수단 프로필 이미지 업로드 및 표시 기능 디자인 스펙

## 1. 개요
성만 FC 공식 홈페이지의 선수단(SQUAD) 목록과 선수 상세 정보 모달에 실제 선수들의 프로필 이미지가 노출될 수 있도록 시스템을 개편합니다. 별도의 이미지 스토리지 서버 없이, 브라우저 환경에서 작동하는 **Base64 Data URL 인코딩 기법**을 활용하여 로컬 스토리지(`squadData`) 내에 프로필 이미지를 직접 보관하고 렌더링하는 클라이언트 사이드 단독 이미지 업로드 아키텍처를 도입합니다.

---

## 2. 데이터 및 업로드 설계

### A. 선수 데이터 스키마 확장
* `squadData` 배열 내의 개별 선수 객체 내 `image` 필드에 Base64 형식의 이미지 데이터 문자열을 할당합니다.
* **데이터 예시 (Base64 인코딩):**
  ```json
  {
    "id": 1,
    "name": "김성민",
    "engName": "KIM Sungmin",
    "number": 10,
    "position": "FW",
    "stats": { "matches": 15, "goals": 9, "assists": 4 },
    "details": { "height": 183, "weight": 78, "birth": "1998-05-12" },
    "image": "data:image/png;base64,iVBORw0KGgoAAAANSU..." // Base64 Data URL 저장
  }
  ```
* **기본값 (Fallback):**
  * 사용자가 별도의 이미지를 설정하지 않은 경우, 기존의 정적 문자열 식별자(예: `player_fw_10`) 혹은 빈 값을 유지합니다. 이 경우 프론트엔드에서는 등번호 기반의 기본 디자인 상자를 노출합니다.

### B. 로컬 이미지 인코딩 흐름 (FileReader API)
* 관리자 선수 추가/수정 폼 내에 이미지 업로드 컨트롤을 구성합니다.
* 파일 변경(`change`) 이벤트 감지 시, `FileReader` 객체의 `readAsDataURL(file)` 메서드를 사용하여 비동기적으로 파일을 Base64 문자열로 변환합니다.
* 변환이 완료되면 폼 내부 미리보기 박스를 갱신하고 상태 변수(`loadedImageData`)에 바인딩합니다.

## 3. UI 및 디자인 설계

### A. 선수단 카드 목록 (`.player-card` 내 렌더링 변경)
* **CSS 클래스 신설 (`.player-img-wrapper`, `.player-img`):**
  * `height: 240px;` 크기를 갖고 카드 상단 영역에 맞게 이미지를 조정하는 래퍼를 생성합니다.
  * 이미지는 `object-fit: cover;` 스타일을 적용해 비틀어짐 현상을 방지합니다.
  * 마우스 호버(`:hover`) 시 이미지가 `scale(1.05)`로 확대되며 전환되는 부드러운 트랜지션 효과를 가미합니다.
* **동적 렌더링 분기:**
  * `player.image` 값이 존재하고 `data:image/` 또는 `http` 형식으로 시작되는 경우 이미지가 있는 템플릿을 그리고, 그렇지 않으면 기존의 `.player-img-placeholder`를 그려 호환성을 완전 유지합니다.

### B. 선수 상세 모달 (`#playerModal` 내 레이아웃 변경)
* 헤더 섹션(`.player-modal-header`)의 왼쪽에 가로 `100px`, 세로 `133px` 비율의 이미지 프레임을 배치합니다.
* 둥근 모서리(`border-radius: 8px`) 및 얇은 테두리(`border: 1px solid var(--color-glass-border)`) 스타일을 부여해 구단 테마와 일체감을 줍니다.
* 사진이 등록된 경우에만 노출하고, 사진이 없는 경우 기존 등번호 모달 스타일이 정렬되도록 조건부 렌더링을 실시합니다.

### C. 관리자 선수 폼 개편 (`showSquadForm`)
* 등번호와 포지션 입력 필드 하단에 이미지 업로드 영역을 구성합니다.
  * **미리보기 박스:** `width: 80px; height: 80px;` 크기의 둥근 사각형. 등록된 사진이 보이며 사진이 없으면 현재 등번호가 가운데 크게 노출됩니다.
  * **업로드 버튼:** 클릭 시 파일 탐색기를 여는 보이지 않는 파일 업로드 컨트롤을 작동시킵니다.
  * **이미지 제거 버튼:** 클릭 시 현재 바인딩된 이미지 변수를 제거하고 미리보기를 등번호 플레이스홀더로 되돌립니다.

## 4. 접근성 및 보안 (a11y & Security)
* 업로드되는 `<input type="file">`에 명확한 스크린 리더용 라벨을 연결합니다.
* 이미지 렌더링 시 대체 텍스트(`alt`)에 선수 이름을 명시하여 시각 장애인을 보조합니다.
* 파일 사이즈 제한: 로컬 스토리지 용량 한계(대략 5MB)를 방어하기 위해 용량이 지나치게 큰 원본 파일(예: 1.5MB 이상) 업로드 시 경고창("1.5MB 이하의 이미지만 업로드 가능합니다.")을 띄우고 차단하는 예외 처리를 적용합니다.
